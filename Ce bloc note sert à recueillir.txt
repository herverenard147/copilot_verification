Ce bloc note sert à recueillir tout ce que nous avons besoin pour la conception de notre plateforme. 

CORD = = Consolidated Receipt Dataset, un jeu de données créé par Clova AI (Naver) pour le "post-OCR parsing" de reçus/tickets de caisse. Chaque image est accompagné par sa transcription faite par un humain

OCR (Optical Character Recognition / Reconnaissance Optique de Caractères) = Il permet de transformer image ( photo d'un texte, scan ) en un fichier texte. Tu lui donnes cette image et il te fait sortir le texte qu'il contient sans comprendre pour autant tout ce qui est ecrit.

Parsing = L'action d'analyser un texte pour en extraire une structure logique. Ici : prendre le texte brut sorti par l'OCR et le découper en champs identifiés (nom du magasin, produit, prix, total...).

Post-OCR Parsing = Vient apres l'OCR pour comprendre le texte en question.

Dataset = Un ensemble de données utilisés pour entraîner ou evaluer un model de machine learning. 

Box-level annotation (annotation au niveau des boîtes) = C est le fait de dessiner une boite sur chaque zone de texte détecter sur l'image pour lui attribuer le texte contenu là dedans.

Labels sémantiques (semantic labels) = En plus de lui donner le texte dans la boîte on lui attribue une signification. En vue de lui donner un contexte

Superclasses / sous-classes = Une hiérarchie de catégories. CORD a 8 catégories larges (superclasses) — ex : informations magasin, menu, sous-total, total, paiement — et chacune se divise en catégories plus fines (54 sous-classes) — ex : dans "menu", tu as "nom du produit", "quantité", "prix unitaire" séparément.

Benchmark = C'est un jeu de données standard utilisé par toute la communauté en vue de tester tout les modéles sur les memes critères. 

Document Understanding / Key Information Extraction (KIE) = C est le domaine de recherche qui vise à faire comprendre à un modèle la structure et le sens d'un document. En gros pas juste lire mais aussi extraire uniquement les parties clés qui nous interessent dans un document.

Modèles multimodaux = Un Modeles multimodal est un modele qui prend en compte plusieurs types/canaux de données. En comparaison avec un unimodal lui se concentre sur un point bien precis: 
Un modèle NLP classique (comme ton DistilBERT sur tweet_eval) est unimodal : il ne voit que du texte. Il ignore complètement la mise en page, les couleurs, les images.
Un modèle de vision classique (classification d'images) est unimodal aussi : il ne voit que des pixels, aucun texte.

Un multimodal lui prend plusieurs canaux:
Le texte — ce que l'OCR a extrait de chaque boîte ("Coca-Cola", "12.50")
La position spatiale — les coordonnées (x, y) de chaque boîte sur l'image (est-ce en haut, en bas, à gauche, aligné avec quoi)
L'image elle-même — parfois même les pixels bruts autour du texte (police, couleur, encadrés, logos)

Token = C'est le faite de decouper des mots, des bouts de mots, des symboles en des petits morceaux

Split = C est le faite de diviser le dataset en 3 ( entrainement 80%/validation 10%/test 10%

Inférence = Le fait d'utiliser un modele deja entraine en vue d'avoir une prédiction.
